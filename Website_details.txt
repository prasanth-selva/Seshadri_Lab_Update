The goal is to set up a new React project with a shadcn/ui structure, TypeScript, and Tailwind CSS, and then integrate a provided 3D interactive component (SplineSceneBasic) which relies on @splinetool/react-spline and various UI components like Spotlight and Card. Setup Instructions provided for missing structure
Since the codebase is empty, it does not support the shadcn structure, Tailwind CSS, or TypeScript yet. Here are the instructions to set them up (which we will execute shortly):

Initialize Project: Run npx create-next-app@latest . --ts --tailwind --eslint --app --src-dir false --import-alias "@/*"
Install shadcn/ui: Run npx shadcn-ui@latest init and choose the default options (default style, neutral color). This automatically creates components/ui for UI files.
Important Note on /components/ui: Using components/ui is crucial because it keeps your core building block components (like Buttons, Cards) isolated from your app-specific business logic components, maintaining a clean and scalable architectural pattern. The default path for styles in Next.js will be app/globals.css, and for components, it will be components/.

Answers to Questions
What data/props will be passed to this component?
SplineScene: Takes scene (a URL string of the Spline scene) and an optional className for styling.
Spotlight (aceternity): Takes className and fill (color string).
Spotlight (ibelick): Takes className, size (number), and springOptions for Framer Motion.
SplineSceneBasic: Currently takes no props. It is self-contained.
Are there any specific state management requirements?
No global state management is needed. The ibelick/spotlight component uses local React state (useState, useRef) and framer-motion hooks (useSpring, useTransform) for cursor tracking.
Are there any required assets (images, icons, etc.)?
The primary asset is a remote 3D scene loaded from https://prod.spline.design/kZDDjO5HuC9GJUM2/scene.splinecode. We will use stock Unsplash images if any image placeholders are needed elsewhere, and lucide-react for any necessary icons.
What is the expected responsive behavior?
The SplineSceneBasic component uses a flexible grid (flex-col / flex-row). It primarily relies on Tailwind's responsiveness. We will apply responsive sizing (text-4xl md:text-5xl, layout shifts using md:left-60) to ensure it scales correctly on mobile devices.
What is the best place to use this component in the app?
This type of immersive 3D component is best utilized in the 'Hero' section on a landing page, or within key highlighted feature sections where capturing the user's attention is crucial. We will implement it in the main app/page.tsx file for demonstration.
